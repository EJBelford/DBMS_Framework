PCD Users Guide

1. Frequently Asked Questions

1.1 What Should I Do If I Cannot Find My Contract/Program?

To create a PCD for a Contract or Program not listed on the main page, contact the PCD Administrator to have it added to the system.

1.2 How Do I Create A New PCD?

To Create a new PCD first choose whichever Contract you want the PCD to apply to. If you want the PCD to apply to multiple contracts, just choose one of them and the others can be added later. If the contract you want your PCD to apply to is not shown, that means it is not yet in the system. Click the link at the bottom of the page to e-mail the PCD Administrator and request that the contract be added to the system. Once you click the desired contract, you will be directed to a page listing all PCD's for that contract. Click the "Create A New PCD" link at the top of that page to create a new PCD.

1.3 How Do I Approve A PCD?

There is a link on the top of the main page labeled "PCD(s) Awaiting Your Approval (X)". The number in the parentheses represents the number of PCD's that are currently awaiting your approval. Clicking on that link will direct you to a page that lists all PCD's awaiting your approval. From there you can click on any PCD to see its details and either approve or rework it.

1.3.1 How Do I Modify A PCD's Program Distribution List?

As an Approver, you can delete individuals off the PCD’s Program distribution list. To remove individuals from this PCD's Program distribution list, go to the program recipients field. Then select the individual that you will need to delete. To select multiple individuals, use the CTRL or the Shift key. Click “Delete”.

1.4 How Do I Find An Existing PCD?

There are two ways to locate an existing PCD. The first is to choose a contract or program that the PCD belongs to from the main page. This will direct you to a page that shows all approved PCDs that were created in the current year for that contract or program. If the PCD has not yet been approved, you can check the box at the top of the screen for the state that the PCD is in and click the "Refresh List" button. This will show all of the current year's PCDs that are in the checked states. To show PCDs that were created in all years and are in the checked states, click the "Show all Years" button. When the "Show all Years" button is clicked, the button label changes to "Show Current Year". At that point, you can click the "Show Current Year" button to see PCDs that were created in the current year and have the checked states.
The second way to find an existing PCD is to search for it. On the main page there is a link on the top of the screen labeled "Search". Clicking on that link will direct you to a page where you can enter and search on any details of a PCD. Clicking the search button will then display a list of all PCDs meeting that criteria.

1.5 How Do I Revise An Existing PCD?

Anyone may submit a revision for a PCD that has been approved. To do this first find the existing PCD using one of the methods described above. When you click on that PCD you will be taken to the View PCD screen. At the bottom of that screen there will be a Revise button. Clicking that button will direct you to the Edit PCD screen which will be pre-filled with the details of the PCD you want to revise. Make any desired changes and click the "Save" button to access the draft at a later date or click submit. If the revision is approved the original PCD will remain in the system, but will reference the revision.

1.6 How Do I Copy An Existing PCD?

To copy a PCD, first find the PCD you want to copy by using one of the methods described above. When you click on that PCD you will be taken to the View PCD screen. At the bottom of that screen there will be a "Copy to another PCD" button. Clicking that button will copy the PCD and direct you to the Edit PCD screen which will be pre-filled with the details of the PCD you wanted to copy. Make any desired changes and then click submit. If the copy is approved it will receive a new PCD number just as any New PCD.

1.7 How Do I Print a PCD?

To print a PCD, click on the "View Printable Version" link on the top right corner of the PCD. After the new window loads, click on the "Print" button on top of the page and click "Close Window" to close the window.

1.8 Are reminder notices sent to the reviewer(s)?

Reminder notices are sent out every weekday to the reviewers who have PCDs awaiting their approval.

1.9 How do I Close a PCD and its Revisions?

There is a link on the top of the main page labeled "PCD(s) Pending Closure (X)". The number in the parentheses represents the number of PCD's that are currently pending closure. Clicking on that link will direct you to a page that lists all PCD's awaiting closure. From there you can click on any PCD to see its details and either close or rework it. To close the PCD, click the "Close PCD" button located on the bottom of the screen. You will have the option of closing just the current PCD or automatically closing the current PCD along with all of its revisions that are still open.

1.10 How do I view archived PCD's for a contract?

There is a link on the top of the main page labeled "View Archive". Clicking on that link will direct you to a page that lists the contracts that are currently archived. After selecting a contract, you can view the PCD's associated to this contract by selecting "View". A list of the PCD's associated with that contract will appear.

2. Overview

     The Program Control Directive (PCD) application handles the generating, routing, approving and filing/archiving of Program Control Directives (PCD’s).  PCD’s are used by the Program Office to direct and authorize personnel to perform actions outside normal circumstances.   Common actions are to procure certain pieces of hardware, loan items to other facilities temporarily, and to stop work on a specific workpackage until requirements are clear.  Currently the principal user is the Lockheed Martin Manassas Navy Systems Program Office.

3. Process Flow

     This section describes the process of creating, editing, submitting, and approving PCD's.  The following is a list of possible states for PCD's:
  • Draft
  • Submitted
  • Rework
  • Approved
  • Closed
  • Archived
     Each of the steps in the process flow is described below:
Creation Creates a new PCD in the Draft state.
Editing Editing can only be performed on PCD's which are in the Draft or Rework state.  The ability to Edit the PCD is restricted to the Originator or an Administrator.
Submission Once a PCD is ready, it can be submitted to the approver(s) for approval.   When the PCD is submitted, each of the approvers receives a notification e-mail indicating which PCD has been submitted to them for approval.   Once submitted, a PCD cannot be modified.
Approval Each assigned approver can edit the PCD.   When all approvers have approved a PCD, its changed to the Approved state. A notification is sent to the program recipient(s), action responsible person, additional recipients, and the originator.  PCD Numbers are generated by the system and are only assigned once a PCD is approved.
Rework There is a place for an approver to enter comments if any (the comments are mostly intended to indicate the reasons for rework if a PCD is designated for rework).   A PCD enters the rework state when any approver has designated the PCD for rework.   A notification e-mail is sent to the originator indicating that the PCD has been designated for rework.
Delete In order to keep a trail, PCD's are never deleted once they have been approved.   However, PCD's in the draft or rework states can be deleted by the Originator or an Administrator.
Revision Once a PCD has been approved it cannot be modified.  In order to allow for updates, revisions can be made to PCD's.  Any user can submit a revision to an approved PCD and initiate the approval process defined above.  If the revision is approved, the original PCD will remain and have a reference to the revised PCD.
Closed Once a PCD has been approved it can be closed.  Only the orginator, approver(s), or Administrator can close a PCD.  After the PCD has been closed it will be noted in the status field and give the id of the user that closed it.
Archived Once a PCD has been closed it can be archived.  Only an Administrator can archive a PCD.   An archived PCD is veiwed using the contract association and can not be modified.

4.  Application Functions

     The application is divided into the following functional parts:  Contracts/Programs, Search, and Administration.  Each piece is accessible from the home page.  The home page initially displays a list of the contracts with a hyperlink to show all PCD's for each contract. The Programs tab may be chosen to change the view to a list of all programs with hyperlinks to show all PCD's for any program.  Links are available for the Search and Administration options, although the Administration button is only available to users with Adminstration authority.
     In addition to the main functions of the application, options are available to display help and to get statistics regarding all PCD's or just the PCD's associated with a specified contract or program.
 

4.1 Contracts/Programs

     Allows users to work with PCD's associated with the selected contract or program.  This option is selected by clicking on the hyperlink for the desired contract or program on the home page.  A list of PCD's associated with that contract or program is displayed.  Initially only approved PCD's will be shown. PCD's of other states can be viewed by selecting the checkboxes for the desired states in the Show box at the top of the screen and clicking the Refresh List button. Selecting the hyperlink for any PCD allows the user to work with the individual PCD. PCD's that have an approved revision will appear with a gray background.

4.1.1 PCD Fields

     When the user selects a PCD, a screen with all the PCD information will be displayed. The following fields may be viewed and edited.
      Table 4-1 PCD Fields  
Field Description
PCD A unique identifier for each PCD. It is automatically generated. For approved PCD's the format is "Last two digits of the year approved-program-company-PCDxxxxx" eg. 01-ARCI-LMDM-PCD00010. For unapproved PCD's the number is in the format "Draft-six digit number" e.g. Draft-000010
Date The latest modification date of the PCD
Department Text box prefilled with department of the originator, but can be changed to any department
Revision This is an uneditable field that shows whether this PCD has had a revision approved. If so, it will show the number of that PCD. If not, it will simply say "Current".
Programs Author may select which program(s) this PCD applies to from a list of all programs that the chosen contract(s) apply to
Originator Uneditable field that shows the name of the PCD's originator
Approver Selected by author from a list of approvers for each contract that a PCD is associated with. One or more approvers must be selected from each Contract that the PCD pertains to. If the same person is an approver for multiple contracts, they may approve for both.
Current Status Uneditable field filled with the current state of the PCD. In the case of rework, approved, or closed the name of the approver(s) who put it in that state and the date they did it will also be displayed
Action Responsible Persons People that are responsible for the action of the PCD. Lookup select by author or a team name that the author enters
Subject The subject of this PCD; Typed by the author
Contracts Shows all contracts or purchase orders that this PCD applies to.
Reference Anything this PCD may be in reference to as entered by the author in a text area.
Action The main content of the PCD
Program Recipients Uneditable list of all program recipients. These are the all people on the program recipient lists of all programs associated with this PCD. They will receive a notification e-mail when the PCD is approved.
Additional Recipients Any additional people that will get notified when the PCD is approved. The author chooses them through Lookup
Attachments Any files can be stored with the PCD by the author
Rework Comments An approver can fill this field out when designating a PCD for rework. It should contain information regarding what changes are necessary
Program to Use in PCD Number The author must select the program to have the system use when it assigns a PCD number to the PCD. To set it the author must select a program from the programs box and click the set code button.

     The modes are available when working with an individual PCD: Edit, Approve, View.  When a PCD has been selected, the mode in which its displayed depends on its state and the authority of the current user.  The application picks which mode to open the PCD in based upon the Conditions column of  Table 4-2.

4.2 Search

     When the search button is pressed, the user is presented with a screen where the search criteria are entered.  Entering data in any of the search fields indicates that the resulting PCD's must contain that criteria.  Wildcards are allowed and are specified using a "*".
     Once the search has completed, a list of all resulting PCD's is displayed.  Hyperlinks are available to each PCD which allow the user to work with the corresponding PCD in the same manner which is available from the Contracts/Programs option.  See Section 4.1 - Contracts/Programs
 

4.3 Administration

     The Administration option allows maintenance of the programs and contracts.  It also allows addition/removal of system administration authorization.  System administrators have access to the Administration option on the home page and are also allowed to edit any PCD in any state.
